# 将来AI画像生成を追加する場合の考え方

現在のPWA版は、AI画像生成APIを使わず、スマホ内のCanvasで「今日の1枚」日記カード画像を作ります。まずはこの機能を使ってください。

## できないこと

- アプリからChatGPTのWeb画面を勝手に操作して画像を作ることはできません。
- GitHub Pagesだけでは、APIキーを安全に隠せません。
- PWAのJavaScriptにAPIキーを書くと、誰でも読めてしまいます。

## AI画像生成に必要なもの

OpenAI APIなどの画像生成APIを使う場合は、次の構成が必要です。

- OpenAI APIなどの画像生成API
- APIキー
- APIキーを安全に持つバックエンドまたはサーバーレス関数
- APIキーを保存する環境変数

## APIキーの置き場所

APIキーはPWAのJavaScriptに直接書かないでください。

`.env.example` を参考に、バックエンドやサーバーレス関数側で `.env` を作ります。

```text
OPENAI_API_KEY=
ALLOWED_ORIGIN=
```

実際の `.env` はGitHubに上げないでください。このリポジトリでは `.gitignore` に `.env` と `.env.*` を入れています。

## 実装するなら

将来AI画像生成を追加する場合は、以下のような流れにします。

1. PWAで日記本文や手書きメモから必要な情報をまとめる。
2. PWAからサーバーレス関数へリクエストを送る。
3. サーバーレス関数が環境変数の `OPENAI_API_KEY` を使って画像生成APIを呼ぶ。
4. 生成された画像URLまたは画像データだけをPWAへ返す。
5. PWAが画像をIndexedDBに保存する。

まずはAPIなしで動くCanvasの日記カード画像化を使うのが安全で簡単です。
